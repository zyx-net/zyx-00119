# 电脑维修接单系统

本地桌面工具，纯前端实现，无需任何后端或数据库。双击即可使用。

---

## 启动方式

### 方式一：直接双击（最简单）
双击 `index.html` 即可在默认浏览器中打开使用。

### 方式二：本地 HTTP 服务器（推荐）
避免某些浏览器的本地文件安全限制，推荐用此方式：
```bash
# 在 computer-repair-tool 目录下执行
python -m http.server 8080
# 然后浏览器访问 http://localhost:8080
```
或
```bash
npx serve .
```

## 快速验证：报价改价 Bug 修复

> **Bug 描述**：同一工单出过一次报价后，修改配件/工时价格再生成新报价，金额仍沿用旧价。
>
> **根因**：`showQuoteModal` 用旧报价的历史价格填充输入框，设置 `select.value` 不触发 `onchange`，导致价格停留在旧值。
>
> **修复（2处，双重保险）**：
> 1. **UI层**：预填充后手动调用 `quotePartChange`/`quoteLaborChange`，从 select 的 `data-price`（最新配置）重算
> 2. **数据层**：`doQuote` 保存时根据 `partId` 从 `Store.getParts()` 实时读最新价，不依赖 DOM

### 一键验证（推荐，30秒出结果）
1. 启动本地服务器（见上方）
2. 浏览器打开 `http://localhost:8080/test-quote-price.html`
3. 点击 **「▶ 运行全部测试」**
4. 看到 **「🎉 全部测试通过（8/8）」** 即表示修复正确

测试覆盖：改价后二次报价、旧版本不串改、刷新一致性、多版本独立、详情页渲染正确等8个用例。

## 数据存储说明

所有数据保存在浏览器 `localStorage` 中，共 8 个独立 key：

| Key | 说明 |
|-----|------|
| `crp_orders` | 工单数据 |
| `crp_parts` | 配件配置 |
| `crp_labor` | 工时配置 |
| `crp_quotes` | 报价版本（**快照存储，永不修改**） |
| `crp_history` | 操作历史时间轴 |
| `crp_rollback_records` | 撤回记录（含原因） |
| `crp_termination_records` | 终止记录（含原因） |
| `crp_initialized` | 样例数据初始化标记 |

**关键保证**：
- 报价版本一旦生成，内部的配件单价和工时费是**独立快照**（JSON深拷贝）
- 修改配件/工时配置**只影响新报价**，历史版本永远保持当时价格
- 关闭浏览器/重启电脑后，所有数据完全一致

## 回归测试（统一口径 · 三端一致）

**核心原则：一套用例定义，三处入口执行，结果格式完全统一。**

所有测试用例定义在 `test-cases.js`，共 **12 个用例**（7 个分类），浏览器页面、Node.js 脚本、Python 脚本使用完全相同的用例 ID、名称、分类和结果统计口径，确保三处数字完全一致。

### 入口一：浏览器回归测试中心（推荐，功能最全）

文件：`test-regression.html`

打开后点击「▶ 运行全部测试」。包含以下功能模块：

| 模块 | 功能 |
|------|------|
| 执行测试 | 12 个用例逐条执行，实时显示状态和日志 |
| 结果恢复 | 页面加载时自动检测上次运行结果，可一键恢复 |
| 历史记录 | 保存最近 50 次运行记录，点击查看详情 |
| 结果对比 | 选择两次运行对比，识别新增失败/已修复/新增/消失用例 |

**数据存储：** 测试结果保存在 `localStorage` 的 `crp_test_results` key 中，与业务数据完全隔离。测试执行使用 `crp_test_*` 命名空间，不影响正式业务数据。

## 技术说明

- **纯前端**：HTML + CSS + 原生 JS，零依赖
- **数据持久化**：localStorage，8个独立 key
- **架构分层**：数据层(Store) → 校验层(Validator) → 状态引擎(StatusEngine) → 报价引擎(QuoteEngine) → UI渲染
- **状态机**：7步正常流程 + 异常终止，共8种状态
- **版本快照**：每个报价版本深拷贝保存，历史永不修改
