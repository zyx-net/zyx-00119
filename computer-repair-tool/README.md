## 快速验证：报价改价 Bug 修复

> **Bug 描述**：同一工单出过一次报价后，修改配件/工时价格再生成新报价，金额仍沿用旧价。
>
> **根因**：`showQuoteModal` 用旧报价的历史价格填充输入框，设置 `select.value` 不触发 `onchange`，导致价格停留在旧值。
>
> **修复（2处，双重保险）**：
> 1. **UI层**：预填充后手动调用 `quotePartChange`/`quoteLaborChange`，从 select 的 `data-price`（最新配置）重算
> 2. **数据层**：`doQuote` 保存时根据 `partId` 从 `Store.getParts()` 实时读最新价，不依赖 DOM

### 一键验证（推荐，30秒出结果）
1. 启动本地服务器（见上方）
2. 浏览器打开 `http://localhost:8080/test-quote-price.html`
3. 点击 **「▶ 运行全部测试」**
4. 看到 **「🎉 全部测试通过（8/8）」** 即表示修复正确

测试覆盖：改价后二次报价、旧版本不串改、刷新一致性、多版本独立、详情页渲染正确等8个用例。

所有数据保存在浏览器 `localStorage` 中，共 8 个独立 key：

| Key | 说明 |
|-----|------|

| `crp_parts` | 配件配置 |

- 报价版本一旦生成，内部的配件单价和工时费是**独立快照**（JSON深拷贝）

- 关闭浏览器/重启电脑后，所有数据完全一致

**核心原则：一套用例定义，三处入口执行，结果格式完全统一，跨重启可恢复，失败可追溯到步骤。**

所有测试用例定义在 `test-cases.js`，共 **12 个用例**（7 个分类），浏览器页面、Node.js 脚本、Python 脚本使用完全相同的用例 ID、名称、分类和结果统计口径，确保三处数字完全一致。

三端（浏览器/Node.js/Python）使用完全一致的：
- **用例定义**：ID、名称、分类、描述、优先级、预期值

- **导入导出格式**：JSON 结构、字段名、版本号完全统一
- **必填字段**：`runId`/`suiteVersion`/`startedAt`/`total`/`passed`/`failed`/`results`

**导入安全机制（三端完全一致）：**
- ✅ **重复识别**：相同 `runId` 自动跳过，不覆盖已有记录，保留历史版本
- ⚠️ **版本冲突**：格式版本不一致时警告并尝试兼容导入，详细报告冲突内容

### 完整链路说明

**执行 → 持久化 → 恢复 → 对比 → 导出 → 导入** 全链路：

1. **执行**：运行测试，逐条记录断言步骤，失败时捕获完整执行轨迹
2. **持久化**：结果自动保存到 `crp_test_results`，页面状态保存到 `crp_test_state`
3. **恢复**：刷新/重启/重新打开页面后，自动检测并提示恢复上次结果
4. **对比**：历史记录中任意两次运行可对比，识别回归和修复
5. **导出**：选择导出全部或单次运行，包含完整日志和步骤信息
6. **导入**：导入时自动去重、校验字段、检测冲突，**冲突时不覆盖旧记录**，保留完整历史

### 验收标准（三端一致）

运行以下三段流程，确认展示、历史和提示完全一致：

1. **执行测试**：12 个用例全部通过，结果保存成功
2. **重启恢复**：关闭页面重新打开，自动检测到上次结果，恢复后数据完全一致
3. **导出再导入**：导出结果 → 清空历史 → 导入文件 → 确认数据完整恢复，无重复、无覆盖、无丢失

## 批量导入审计中心

### 入口
左侧导航栏 → 「📋 批量导入审计」

### 功能概述
批量导入审计中心是一个完整的数据导入管理模块，支持工单、报价、历史记录的批量导入，提供完整的预检、导入、回滚、导出和审计追踪能力。

### 核心特性

#### 1. 独立入口与界面
- **任务列表**：查看所有导入任务，支持按状态、来源文件、处理人筛选
- **新建导入**：上传 JSON 文件，自动完成数据预检
- **任务详情**：查看完整的任务信息、导入结果、冲突明细

#### 2. 导入数据类型
支持导入以下三种数据类型（可混合或单独导入）：
- **工单 (orders)**：维修工单基础数据
- **报价 (quotes)**：报价版本快照数据
- **历史记录 (history)**：工单状态变更历史

#### 3. 预检机制（导入前自动执行）
上传文件后自动完成以下检查：
- **数据类型检测**：自动识别包含的数据类型
- **必填字段校验**：检查所有必填字段是否完整
- **重复数据检测**：检测 ID 或工单号重复
- **版本冲突检测**：检测同一工单相同版本的报价
- **关联关系校验**：检测报价/历史关联的工单是否存在
- **统计报告**：显示可导入数、冲突数、错误数

#### 4. 冲突检测与提示（不覆盖原则）
| 冲突类型 | 说明 | 处理方式 |
|---------|------|---------|
| **重复数据** | ID 或工单号已存在 | 自动跳过，保留旧记录 |
| **版本不一致** | 同一工单已存在相同版本报价 | 自动跳过，保留旧记录 |
| **缺少字段** | 必填字段为空或缺失 | 标记为错误，无法导入 |
| **权限/关联** | 报价/历史关联的工单不存在 | 自动跳过，无法导入 |

**重要保证**：所有冲突项**坚决不覆盖旧记录**，明确提示冲突原因。

#### 5. 审计记录
每次导入任务完整记录：
- **来源信息**：文件名、文件大小、上传时间
- **处理信息**：处理人、操作备注、创建时间
- **执行信息**：开始时间、完成时间、执行状态
- **结果统计**：导入数、跳过数、错误数（分工单、报价、历史）
- **冲突明细**：所有冲突项的类型、严重程度、详细说明
- **快照数据**：导入前后的数据快照，用于回滚

#### 6. 回滚功能
- 支持对已完成的导入任务执行回滚
- 一键恢复到导入前的状态
- 回滚时记录处理人、原因、时间
- 已回滚的任务标记为「已回滚」状态，不可重复回滚

#### 7. 导出处理结果
- 导出本次导入的完整处理结果
- 包含任务信息、统计数据、冲突明细
- 格式：`crp-import-result` v1.0.0
- 可用于存档或二次导入参考

#### 8. 状态持久化与恢复
- **任务状态**：所有导入任务持久化存储
- **筛选条件**：筛选器状态自动保存
- **页面位置**：当前标签页、选中任务自动保存
- **跨重启恢复**：刷新或重启后自动恢复上次状态

**自动恢复链路：**
1. 页面加载时读取 `crp_import_state`
2. 恢复上次停留的标签页（列表/新建/详情）
3. 恢复筛选条件（状态、来源、处理人）
4. 恢复上次选中的任务详情
5. 任务列表数据完全一致

### 使用限制
1. **文件格式**：仅支持 JSON 格式
2. **文件大小**：建议单文件不超过 10MB
3. **数据量**：单次导入建议不超过 1000 条
4. **权限控制**：本工具为本地桌面版，无用户权限控制，导入前请确认数据来源
5. **回滚时效**：任务创建后任意时间可回滚，但建议在发现问题后尽快执行
6. **快照保留**：每个任务保留导入前后的数据快照，请注意存储空间

### 排查办法

#### 常见问题 1：上传文件无反应
- **检查**：文件格式是否为 JSON
- **检查**：文件是否损坏或内容为空
- **检查**：浏览器控制台是否有错误信息

#### 常见问题 2：预检不通过，无法导入
- **查看**：预检结果中的错误详情
- **检查**：是否有缺少必填字段的记录
- **检查**：是否有关联工单不存在的报价/历史
- **解决**：修复数据后重新上传

#### 常见问题 3：导入后数据数量不对
- **查看**：任务详情中的「跳过数」
- **检查**：冲突明细，了解跳过原因
- **常见原因**：重复 ID、重复工单号、版本冲突

#### 常见问题 4：回滚失败
- **检查**：任务状态是否为「已回滚」（不可重复回滚）
- **检查**：是否有快照数据（导入失败的任务可能无快照）
- **检查**：浏览器控制台是否有错误

#### 常见问题 5：重启后状态未恢复
- **检查**：浏览器是否禁用了 localStorage
- **检查**：是否使用了无痕/隐私浏览模式
- **检查**：是否清理了浏览器数据

#### 常见问题 6：导入数据后原数据丢失
- **解决**：立即找到对应的导入任务，执行「回滚」操作
- **预防**：导入前建议先导出原有数据备份
- **预防**：先在测试环境验证导入文件

### 操作流程

#### 标准导入流程
1. 点击左侧导航「📋 批量导入审计」
2. 切换到「📤 新建导入」标签
3. 点击上传区域选择 JSON 文件
4. 等待预检完成，查看预检结果
5. （可选）填写处理人和操作备注
6. 点击「✅ 确认导入」
7. 等待导入完成，自动跳转到任务列表
8. 点击任务查看详情，或进行回滚/导出操作

#### 回滚流程
1. 在任务列表中找到需要回滚的任务
2. 点击「↩ 回滚」按钮
3. 填写处理人和回滚原因
4. 点击「确认回滚」
5. 任务状态变为「已回滚」，数据恢复到导入前

### 数据安全
- 所有数据仅存储在本地浏览器 localStorage
- 导入前自动保存数据快照，确保可回滚
- 冲突数据坚决不覆盖，保护现有数据
- 建议定期导出数据备份
