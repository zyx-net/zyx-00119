🔍 根因与修复说明

根因：同一工单重复报价时，弹窗用旧报价的历史价格填充输入框，设置 select.value 不触发 onchange，导致价格停留在旧值。

修复点1（UI层）：showQuoteModal 预填充后，手动调用 quotePartChange/quoteLaborChange 从 select 的 data-price（最新配置）重算价格。

修复点2（数据层，双重保险）：doQuote 保存时，根据 partId/laborItemId 从 Store.getParts()/getLabor() 实时读取最新价格，不依赖 DOM 输入框。

版本快照：每个报价版本独立存储（JSON深拷贝），修改配置不影响历史版本。
